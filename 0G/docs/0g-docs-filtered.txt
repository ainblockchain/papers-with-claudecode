üíæ Module 3: 0G Storage Integration
0G Storage is a decentralized network designed for massive AI datasets, offering 95% lower costs than AWS and 200 MBPS retrieval speeds.


Storage Types: Supports both an Immutable Log Layer (for ML datasets, archives) and a Mutable Key-Value Layer (for dynamic content, state storage).
+1


TypeScript SDK Installation: npm install @0glabs/0g-ts-sdk ethers.

Uploading a File (TypeScript):

JavaScript
import { ZgFile, Indexer } from "@0glabs/0g-ts-sdk";
import { ethers } from "ethers";

const provider = new ethers.JsonRpcProvider("https://evmrpc-testnet.0g.ai");
const signer = new ethers.Wallet("YOUR_PRIVATE_KEY", provider);
const indexer = new Indexer("https://indexer-storage-testnet-turbo.0g.ai");

const file = await ZgFile.fromFilePath("/path/to/file");
const [tree, treeErr] = await file.merkleTree();
console.log("Root Hash:", tree?.rootHash()); // Save this to download later!

const [tx, uploadErr] = await indexer.upload(file, "https://evmrpc-testnet.0g.ai", signer);
await file.close();
``` [cite: 11, 787, 792, 793, 794]

Downloading a File: Use the indexer.download(rootHash, outputPath, true) method, where true enables Merkle proof verification.

üß† Module 4: 0G Compute (Inference & Fine-Tuning)
0G Compute provides a decentralized GPU marketplace offering 90% cheaper AI workloads. It features pay-per-use pricing and smart contract escrow for trustless payments.
+2

1. Account Funding Flow:
Before making API calls, developers must fund their accounts.
+1


Deposit: Transfer funds from your wallet to your Main Account.


Transfer: Move funds from the Main Account to a Provider Sub-Account (specify whether it is for inference or fine-tuning).
+1

2. Inference (OpenAI SDK Compatible):
0G Compute acts as a drop-in replacement for the OpenAI SDK.
+1


CLI Setup: 0g-compute-cli inference get-secret --provider <PROVIDER_ADDRESS> generates a Bearer token (app-sk-<SECRET>).
+1

Python Integration:

Python
from openai import OpenAI
# Change base_url to the 0G provider and use the generated secret
client = OpenAI(
    api_key="app-sk-<YOUR_SECRET>",
    base_url="<PROVIDER_URL>/v1/proxy"
)
response = client.chat.completions.create(
    model="<model_name>",
    messages=[{"role": "user", "content": "Hello!"}]
)

Data must be prepared in .jsonl format (e.g., {"messages": [{"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]}).

If your learning tool focuses on Web3 gaming or AI Agents, INFTs are a massive differentiator.


Concept: Traditional NFTs just point to static metadata, but INFTs (ERC-7857) contain encrypted AI intelligence that transfers with the token.

Privacy-First: The AI model data is stored encrypted via 0G Storage. When the NFT changes hands, a secure oracle (via TEE or ZKP) re-encrypts the key so only the new owner can access the AI agent's logic.
+1


Use Cases: AI Trading bots, personal assistants, intelligent gaming NPCs, and AI-as-a-service.
